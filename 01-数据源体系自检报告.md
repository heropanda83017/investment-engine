## 二、分层架构（推荐铺排）

```
┌─────────────────────────────────────────────────────────────┐
│                    Layer 1: 实时盘面                          │
│  wudao MCP (29端点) ← 第一优先级，A股一切实时数据从这里走        │
│  TickFlow (盘中逐笔) ← 补充wudao未覆盖的逐笔级别数据           │
├─────────────────────────────────────────────────────────────┤
│                    Layer 2: 结构化数据                        │
│  baostock (32ms) ← K线首选，任何A股K线请求先走baostock         │
│  Tushare (批量财务) ← 全市场筛选/财务分析时启用                 │
│  akshare (基金/期货/宏观) ← 非A股资产类数据                    │
├─────────────────────────────────────────────────────────────┤
│                    Layer 3: 搜索与情报                        │
│  AnySearch MCP ← 主搜索通道(沙盒通用+中文强+垂域)              │
│    ├── search (通用/垂域搜索)                                  │
│    └── batch_search (并行2-5查询)                              │
│  DuckDuckGo ← 本机降级备份(仅本机环境)                         │
├─────────────────────────────────────────────────────────────┤
│                    Layer 4: 资讯与舆情                        │
│  financial-news skill (财经快讯/公告/雪球)                     │
│  aihot/ai-news skill (AI行业动态)                            │
├─────────────────────────────────────────────────────────────┤
│                    Layer 5: 扩展(待接入)                      │
│  StockStar MCP ← ESG评级/调研/DCF (Key到位后启用)             │
│  FinanceMCP ← 分钟K线/美股替代 (第三方不稳定时备选)            │
└─────────────────────────────────────────────────────────────┘
```

## 三、数据流决策树

### 场景1: "查某只A股"
```
wudao MCP (实时行情+资金流+龙虎榜)
  → 还需要K线? → baostock (32ms)
  → 还需要财务? → Tushare/akshare
  → 还需要新闻? → financial-news skill + AnySearch search
```

### 场景2: "全市场筛选"
```
Tushare (0.4s全A股日线)
  → blackhorse-ai因子评分 (6因子+百分位排名)
  → baostock补K线
  → AnySearch补行业新闻
```

### 场景3: "分析某行业"
```
AnySearch垂域 (domain=finance) + 通用搜索
  → financial-news skill (行业快讯)
  → akshare (行业指数)
  → wudao concept_ranking (题材热度)
```

### 场景4: "AI投资分析"
```
aihot/ai-news (行业动态)
  → AnySearch (企业/技术搜索)
  → wudao (相关A股实时数据)
  → Tushare/baostock (财务验证)
```

## 四、通道间降级链

```
K线:      wudao → baostock(32ms) → Tushare(150ms) → akshare(830ms) → 缓存
财务:     Tushare → baostock(270ms) → akshare(420ms) → 缓存
实时快照:  wudao → TickFlow(176ms) → 热缓存(5s TTL)
搜索:     AnySearch(云端) → DuckDuckGo(本机) → 缓存
美股:     FinanceMCP(云端) → yfinance(本机,IP封禁)
```

## 六、关键风险

1. **Tushare API Key 在沙盒不可用** — 批量财务分析需本机运行
2. **AnySearch 约30次/窗口限频** — 正常Agent对话不会触发，批量任务注意间隔
3. **FinanceMCP 是HTTP非加密+个人服务** — 不应用于生产级决策
4. **StockStar Key 未申请** — 独家ESG数据无法获取
5. **美股数据无稳定通道** — FinanceMCP可用但质量待验证，yfinance被Yahoo封禁
